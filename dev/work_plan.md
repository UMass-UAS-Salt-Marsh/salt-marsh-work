# Work plan

Scratch space for the currently active plan. One plan at a time — this
file gets overwritten when new work begins, and is cleared back to this
placeholder once the work lands. The durable record of what was done
lives in `worklog.md`.

A good plan states the goal, lists concrete steps with file/line
references, calls out decision points, and notes trade-offs for any
non-obvious choice. Get sign-off before implementing.
